# Agent 템플릿

> 모든 Agent 파일은 이 형식을 따릅니다.

## 표준 구조

```markdown
# {에이전트명} 에이전트

{한 줄 설명}

## 역할

{구체적인 역할 설명 - 2~3문장}

## 입력

```json
{
  "field": "설명"
}
```

## 처리 규칙

1. {규칙1}
2. {규칙2}
3. {규칙3}

## 출력

```json
{
  "field": "설명"
}
```
```

---

## 필수 섹션

| 섹션 | 필수 | 설명 |
|------|------|------|
| 에이전트명 | O | `# {이름} 에이전트` 형식 |
| 역할 | O | 이 에이전트가 하는 일 |
| 입력 | O | JSON 스키마 |
| 처리 규칙 | O | 번호 목록으로 명확하게 |
| 출력 | O | JSON 스키마 |

## 선택 섹션

| 섹션 | 용도 |
|------|------|
| 사용자 | 라벨별 에이전트에서 개인화 정보 |
| 우선순위 | 특정 주제 강조 |
| 예외 처리 | 엣지 케이스 대응 |

---

## 작성 원칙

1. **이모지 사용 금지**
2. **간결하게**: 50~100줄 이내
3. **명확한 규칙**: 애매한 표현 금지
4. **JSON 스키마 필수**: 입출력 형식 명시
5. **예시는 최소한**: 1개면 충분, 없어도 됨

---

## 라벨별 에이전트 추가 규칙

라벨별 에이전트(`agents/labels/*.md`)는 다음을 추가로 포함:

```markdown
## 사용자

- **역할**: {직업}
- **관심사**: {관련 관심사}

## 우선순위

다음 주제를 우선 추출: {주제1}, {주제2}, ...
```
