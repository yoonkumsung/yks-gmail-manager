# SKILL 템플릿

> 모든 SKILL 파일은 이 형식을 따릅니다.

## 표준 구조

```markdown
# SKILL_{id}

## 메타데이터

| 항목 | 값 |
|------|-----|
| 이름 | {뉴스레터명} |
| 발신자 | {email} |
| 유형 | single-item / multi-item |
| 언어 | ko / en / mixed |

---

## 구조 분석

### 아이템 경계

- 패턴: {구분자 설명}
- 평균 아이템 수: {N}개

### 제목 위치

- 태그: {h1, h2, strong 등}
- 특징: {설명}

### 본문 위치

- 태그: {p, div 등}
- 특징: {설명}

---

## 추출 규칙

1. {규칙1}
2. {규칙2}

---

## 필터링

### 제외 영역

- {영역1}: {이유}
- {영역2}: {이유}

### 제외 키워드

- {키워드1}, {키워드2}, ...
```

---

## 필수 섹션

| 섹션 | 필수 | 설명 |
|------|------|------|
| 메타데이터 | O | 테이블 형식 |
| 구조 분석 | O | 아이템 경계, 제목, 본문 위치 |
| 추출 규칙 | O | 번호 목록 |
| 필터링 | O | 제외 영역과 키워드 |

## 선택 섹션

| 섹션 | 용도 |
|------|------|
| 인코딩 처리 | 특수 인코딩이 필요한 경우 |
| 예시 | 실제 HTML 구조 샘플 (최소한만) |

---

## 작성 원칙

1. **이모지 사용 금지**
2. **테이블 활용**: 메타데이터는 테이블로
3. **구체적 태그명**: "제목 태그" 대신 "h2, strong"
4. **측정 가능한 기준**: "짧게" 대신 "150자 이내"
