## 입력

```json
{
  "message_id": "메시지 ID",
  "from": "발신자",
  "subject": "제목",
  "clean_text": "구조화된 마크다운 본문 (헤딩/볼드/링크/섹션구분 포함)"
}
```

## 입력 형식 안내

clean_text는 **구조화된 마크다운**입니다:
- `## 제목`, `### 소제목` → 뉴스레터 내 아이템 경계 힌트
- `**볼드 텍스트**` → 핵심 키워드/제목 후보
- `[텍스트](URL)` → 원문 링크
- `---SECTION_BREAK---` → 뉴스레터 내 섹션 구분선
- `[IMAGE: 설명]` → 이미지 설명
- `| 컬럼 | 컬럼 |` → 데이터 테이블

이 마크다운 구조를 활용하여 아이템 경계를 정확하게 파악하세요.

**[중요] 원문 기사 전문 블록**: 본문 하단에 `=== 원문 기사 전문 ===` 구분선이 있으면, 그 아래는 fetch가 각 원문 링크를 따라가 가져온 **정식 기사 전문**입니다(보통 `--- <URL> ---` 헤더로 기사별 구분). 해당 아이템의 **요약은 상단 티저가 아니라 이 전문의 핵심사실·수치·인용으로 충실히 작성**하고(티저만으로 끝내지 말 것), **link도 이 전문 블록의 정식 URL을 우선** 사용하세요. 단 전문 하단의 "관련 기사/많이 본 뉴스/인기 기사/추천" 사이드바·푸터는 본 기사가 아니므로 아이템화하지 마세요.

## 처리 규칙

1. **빠짐없이 추출**: 광고/푸터/구독안내만 제외하고 원문의 모든 뉴스 아이템을 추출. 중요도와 무관하게 전부 추출. 하나라도 누락 금지. **메일의 모든 섹션을 끝까지** 처리할 것 — 후반·하단의 목록형 헤드라인 묶음(예: "더 보기", "그 외 소식", 영문/스페인어 추가 섹션)도 빠뜨리지 말 것. 긴 메일이라 청크로 나뉘어도 각 청크의 모든 아이템을 추출.
2. **요약**: 원문 분량에 비례하여 작성.
   - **원문이 충분한 경우 (3문장 이상)**: 200~500자로 요약
   - **원문이 짧은 경우 (1~2문장)**: 원문 수준의 요약 허용 (50~200자도 OK). 짧은 원문을 억지로 200자 이상으로 부풀리지 말 것
   - **긴 콘텐츠 (단일 주제 심층)**: 400~800자
   - 필수 3요소: **WHO(주체)** + **WHAT(핵심 사실)** + **구체적 근거(수치/데이터/사례)**
   - 원문에 수치가 있으면 **반드시 포함** (금액, 비율, 날짜, 규모 등). 통화기호·자릿수·단위를 **원문 그대로 무손실 복제**하고 생략·반올림·자릿수 누락 금지 (예: `$5.55 billion`→"55억 5천만 달러($5.55 billion)", `$185`→"185달러"; 절대 ".55억"처럼 앞자리를 빠뜨리지 말 것)
   - 원문에 수치가 없으면 **구체적 사실과 배경맥락**으로 채움
   - 원문에 시사점이 명시된 경우만 시사점 포함
4. **요약 금지 사항**:
   - 제목을 반복하거나 재서술하는 것으로 요약을 대체하지 말 것
   - "~에 대해 보도했다", "~가 화제다" 같은 메타 서술 금지
   - 원문에 없는 전망/시사점/영향 예측 추가 금지
5. **긴 콘텐츠**: SKILL_작성규칙의 "긴 콘텐츠 처리" 규칙 적용
6. **제목**: 20~50자, 주어+동사+핵심정보
7. **키워드**: 3~5개, 명사형 (고유명사 우선)
8. **금지**: 이모지, 미완성 문장, 원문 복붙, 자의적 추론
9. **링크**: 마크다운 링크 `[텍스트](URL)` 형식에서 **URL만** 추출(마크다운 `[..](..)` 전체를 link에 넣지 말 것). 없으면 빈 문자열. 본문에 `=== 원문 기사 전문 ===` 블록이 있으면 그 블록의 정식 기사 URL(`--- <URL> ---` 헤더 등)을 추적 리다이렉트 링크보다 **우선** 사용.
10. **단일 콘텐츠 뉴스레터**: 뉴스레터 전체가 하나의 주제를 심층 분석하는 경우, 1개의 아이템으로 추출하되 요약을 400~800자로 작성 (구조: 핵심주장 → 근거/데이터 → 결론)
11. **자기검증 (추출 완료 후 필수)**:
    - 원문과 대조하여 누락된 아이템이 없는지 확인
    - 각 요약이 원문 분량에 비례하는지 확인 (짧은 원문은 짧은 요약 OK, 긴 원문은 200자 이상)
    - 각 요약이 제목의 단순 반복이 아닌지 확인
    - 원문의 수치/데이터가 요약에 포함되었는지 확인

## 출력

```json
{
  "items": [
    {
      "title": "간결하고 구체적인 제목 (20~50자)",
      "summary": "핵심 내용 요약 (200~500자, 단일콘텐츠는 400~800자)",
      "keywords": ["키워드1", "키워드2", "키워드3"],
      "link": "원문 URL (없으면 빈 문자열)",
      "source": "뉴스레터 이름"
    }
  ]
}
```
