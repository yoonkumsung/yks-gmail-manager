## 구조 분석

### 뉴스레터 특징
이 발신자는 **두 가지 형식을 섞어 발송**한다. 매 메일마다 형식을 먼저 판별한다.

- **(a) 단일 에세이형(single-topic)**: 하나의 주제를 심층 분석. 기술 비용 하락에 따른 산업 변화 등. 아이템 1개.
- **(b) 로드업형(roundup, "TOP N 많이 읽은 글")**: 메일 제목이 "지난주/이번 주 구독자들이 가장 많이 읽은 글 N편" 형태. 본문 상단 "### **이번 주 독자들이 가장 많이 읽은 글 TOP N**" 아래에 실제 소개 기사(번호가 붙은 `### **1.** … ### **2.** …`)만 뉴스 아이템으로 추출. 이때 각 소개 기사에 딸린 하이퍼링크(예: `news.aikoreacommunity.com/gary-tan-...`)를 그 아이템의 link로 채운다.

### 아이템 경계
- 주제별 소제목으로 구분 (예: '인퍼런스 경제학', '생각하는 비용이 사라진 시대')

### 제목 위치
- 메일 제목과 본문 초반에 명시

### 본문 위치
- 소제목 아래 p 태그 형식의 본문

### 링크 위치
- 참고 자료 섹션에 URL 포함

## 제외 영역

- 구독 관리 섹션
- 관련 기사 추천 영역
- **[중요] 지난주/지난호 "많이 읽은 글·추천 글" 목록 섹션, 뉴스레터 자기참조 링크 목록은 뉴스 아이템으로 추출하지 말 것.** 구체적으로:
  - `### Keep reading` 아래에 나열된 지난 발행분 링크 목록
  - `news.aikoreacommunity.com/r/...`(추적 리다이렉트) 형태의 자기참조 링크만 있고 실제 소개 문구가 없는 목록 항목
  - 메일 최하단 아카이브/과거 발행분 나열(날짜 + "Paid/Members/Public" 라벨이 붙은 카드들, `=== 원문 기사 전문 ===` 이후에 크롤링으로 섞여 들어온 아카이브 목록 포함)
  - 로드업형에서 "TOP N" 아래 **번호가 붙은 실제 소개 기사만** 아이템으로 삼고, 그 외 본문에 박힌 링크 목록/추천 목록은 전부 제외한다. (과추출 방지)
