# SKILL: 바이라인네트워크

IT/테크 전문 미디어의 일간 뉴스레터.

---

## 메타 정보

```yaml
from_patterns:
  - "byline@byline.network"
  - "byline.network"
  - "바이라인네트워크"
type: daily_digest / advertisement
language: ko
frequency: daily
```

---

## 뉴스레터 유형 구분

### 1. 일반 뉴스 (일간 바이라인)
- 제목에 "[일간 바이라인]" 포함
- 여러 짧은 뉴스 아이템

### 2. 광고성 메일
- 제목에 "(광고)" 포함
- 특정 제품/서비스 소개
- **별도 판단 필요**

---

## 광고 판단 기준

### 제목에 "(광고)" 있는 경우

```
제목: (광고) 스페셜리포트 - 데이터독 고객사례 모음

판단:
1. 본문에 기술 정보가 있는가?
   - 있음: 기술 사례 소개로 추출 고려
   - 없음: 순수 광고로 제외

2. 읽는 사람에게 학습 가치가 있는가?
   - 있음: 사례 분석으로 추출
   - 없음: 제외
```

### 예시

**제외 (순수 광고)**
```
"지금 구매하세요"
"할인 혜택"
"무료 체험 신청"
```

**포함 고려 (기술 정보 있음)**
```
"크래프톤 PUBG의 Datadog 기반 장애 대응 프로세스"
"KT: GPU 운영 자동화 사례"
→ 기술적 인사이트 있으면 추출 가능
```

## 추출 규칙

### 일반 뉴스레터
| 유형 | 처리 |
|------|------|
| 각 뉴스 아이템 | 개별 추출 |
| 소개가 충분한 경우 | 그대로 사용 |
| 소개가 짧은 경우 | 제목 + 소개로 요약 |

### 광고성 메일
| 유형 | 처리 |
|------|------|
| 기술 사례 있음 | 사례 내용 추출 |
| 순수 제품 홍보 | 제외 |
| 다운로드 링크만 | 제외 |

---

## 아이템 경계

### 신호
- 빈 줄로 구분된 섹션
- 새로운 제목 (굵게 또는 대괄호)
- 링크가 끝나고 새 내용 시작
