## 핵심 특징: 단일 주제 심층 분석

미라클레터는 **하루 하나의 주제**를 여러 섹션으로 나눠 심층 분석함.

### 아이템 추출 전략
```
전체 뉴스레터 = 1개 아이템

섹션들 (공포가 만든 2배 투자, 챗봇은 이제 안녕?, 한국 기업이 하는 착각)
→ 모두 같은 주제 (BCG AI 보고서)의 다른 측면
→ 하나의 아이템으로 통합
```

## 추출 규칙

### 포함
| 유형 | 처리 |
|------|------|
| 본론 전체 | 하나의 아이템 |
| 3줄 요약 | summary 작성 시 참고 |

### 제외
| 유형 | 예시 |
|------|------|
| 헤더 | "미라클 모닝을 하는 일잘러들의 참고서" |
| 인사말 | "독자 여러분 안녕하세요" |
| 레터 읽는 법 | "볼딕, 밑줄..." |
| 웹에서 보기 | "이 메일이 잘 안보이시나요?" |
| 마무리 인사 | "다음에 뵙겠습니다" |
| 공유 버튼 | "[이미지: 트윗하기]" |
| 이벤트 | 책 이벤트, 5행시 이벤트 (내용 없는 경우) |

### 이벤트 판단
```
이벤트 내용만 있고 뉴스 가치 없음 → 제외
예: "5행시 이벤트 참여하세요" → 제외

이벤트가 뉴스 주제와 연관 → 포함 고려
예: BCG 보고서 관련 책 소개 (책 내용 설명 있으면) → 포함 가능
```

## 장문 판단

미라클레터는 보통 **전체 본론이 1,500자 이상**이므로:

```
is_longform: true (대부분의 경우)
summary: 300~700자 요약
original_text: 본론 전체 (헤더/푸터 제외)
```

## 3줄 요약 활용

미라클레터의 "오늘의 3줄 요약"은 핵심 포인트를 잘 정리해둠.

### 활용 방법
```
1. 3줄 요약을 먼저 읽음
2. summary 작성 시 이 포인트들을 반영
3. 본론에서 세부 내용 보완
```

### 예시
```
3줄 요약:
1. CEO들은 생존 공포 때문에 거품 우려에도 AI 투자를 2배로 늘립니다.
2. 기업 예산은 챗봇이 아닌, 스스로 일하는 'AI 에이전트'로 쏠립니다.
3. 한국 기업은 '챗봇 채택률'을 높이려 하지만 글로벌은 이미 다른 게임을 합니다.

→ summary 작성 시 이 3가지 포인트 반드시 포함
```
