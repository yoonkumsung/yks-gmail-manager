## 구조 분석

### 뉴스레터 특징
- IT/테크 분야 심층 뉴스레터 (한국어)
- 이모지 헤더로 기사 구분이 명확함
- 상단에 목차(제목 리스트) → 하단에 상세 기사
- 유료 콘텐츠 일부 포함

### 이모지 기사 구분 패턴
- 🎯 — 메인 헤드라인 기사
- 🛒 — 기업/비즈니스 전략 기사
- 🚀 — 기술/제품 심층 분석
- 💡 — [더 읽기] 추가 뉴스/인사이트
- ⚡ — 이벤트/컨퍼런스 안내 (보통 광고)
- 🖥️ — 광고/프로모션
- 🎬 — 커뮤니티/멤버십 안내

### 뉴스 vs 광고 구분
- 뉴스: 🎯, 🛒, 🚀, 💡 — **모두 추출**
- 광고: ⚡(이벤트 홍보), 🖥️(브랜딩 광고), 🎬(멤버십 안내) — **제외**

### 아이템 경계
- 이모지 + 제목 줄이 각 기사의 시작
- 다음 이모지 제목까지가 하나의 기사

### 링크 위치
- 기사 내 대괄호 URL 또는 "더 읽기" 링크

---

## 추출 규칙

1. 🎯, 🛒, 🚀, 💡 이모지로 시작하는 기사를 **모두 빠짐없이** 추출
2. ⚡, 🖥️, 🎬는 광고/프로모션이므로 제외
3. 메인 기사(🎯, 🛒, 🚀): 300~500자 요약
4. [더 읽기](💡): 짧은 요약 허용
5. 제목 20~50자, 키워드 3~5개
6. 원문 URL 추출

---

## 제외 영역

- 상단 목차 (기사 시작 전의 제목 나열)
- ⚡ 이벤트/컨퍼런스 홍보
- 🖥️ 광고 배너
- 🎬 멤버십/커뮤니티 안내
- 하단 구독/수신거부

---

## 특이사항

- 유료 콘텐츠 표시가 있으면 "(유료 구독 콘텐츠)" 표기
- 상단 목차와 하단 상세 기사가 중복될 수 있음 → 상세 기사만 추출
- 이모지 패턴이 일관적이므로 섹션 기반 분할에 적합
