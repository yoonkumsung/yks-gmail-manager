## 구조 분석

### 뉴스레터 특징
- 상단에 "간추린 정치·사회", "간추린 경제" 등 카테고리별 짧은 헤드라인 목록
- 하단에 "깊게 보는" 섹션으로 2~3개 메인 기사를 상세 서술
- 각 메인 기사에 읽기 소요시간 표시 (예: "6분 34초")
- 구어체 + 분석적 문체

### 아이템 경계
- 간추린 섹션: ❙ 기호와 카테고리명으로 시작, 개별 뉴스는 줄바꿈으로 구분
- 깊게 보는 섹션: 큰 제목 + 부제목 + 여러 문단의 상세 본문

### 제목 위치
- 간추린 항목: 각 줄 자체가 제목
- 깊게 보는 기사: 굵은 텍스트 또는 큰 글씨 제목

### 본문 위치
- 간추린 항목: 제목만 있고 본문 없음 (제목에서 핵심 사실 추출)
- 깊게 보는 기사: 제목 아래 여러 문단으로 상세 분석

### 링크 위치
- 명시적 원문 링크 없음 (뉴스레터 자체가 원본 콘텐츠)

## 추출 규칙

1. "간추린" 섹션의 각 헤드라인을 개별 아이템으로 추출 (1줄 제목 → 요약은 제목 기반으로 확장)
2. "깊게 보는" 섹션의 메인 기사를 별도 아이템으로 추출 (상세 요약)
3. 제목은 20~50자
4. 간추린 항목: 요약 300자 내외 (제목에서 핵심 사실 추출)
5. 깊게 보는 기사: 요약 300~500자 (핵심사실-수치-배경-시사점)
6. 키워드 3~5개, 명사형
7. 링크는 빈 문자열 (자체 콘텐츠)
