## 구조 분석

### 뉴스레터 특징

- 하나의 메일 안에 두 가지 레이어가 공존
  1. **상단 개요(목차)** — `❙` 기호로 시작하는 1줄 불릿 목록 (본문과 중복되는 요약 메타)
  2. **본문 섹션** — 실제 콘텐츠. 헤더 포맷이 `"TOPIC | MM분 SS초"` (예: `정치·사회 | 8분 41초`, `경제 | 5분 17초`)
- 본문 섹션은 두 종류
  - **깊게 보는 ...** : 단일 심층 기사 (수천 자 분량)
  - **더하기+ TOPIC | MM분 SS초** : 간추린 뉴스 여러 개를 묶은 섹션 (각 아이템에 굵은 소제목 + 본문)

### 아이템 경계

- 본문 헤더 패턴 `^(더하기\+\s*)?[가-힣·\s]+ \| \d+분(\s*\d+초)?$` 을 기준으로 분할
- 한 심층 기사 섹션 = 아이템 1개
- 한 "더하기+" 섹션 = 그 안의 소제목 개수만큼 다수 아이템

### 제목 위치

- 심층 기사: 헤더("정치·사회 | 8분 41초") 바로 아래 1~3줄의 굵은 부제목(예: "미국-이란 합의 / 흘러가는 상황 파악하기")을 제목으로 사용
- 더하기+ 섹션 내 개별 아이템: 굵은 소제목(예: "박상용 검사 피의자 입건 / 2차 특검, 출국금지 조치")을 제목으로 사용

### 본문 위치

- 심층 기사: 부제목 이후부터 다음 본문 헤더(다음 `TOPIC | 시간` 또는 `더하기+ ...`) 또는 "VIP 독자는" 푸터 직전까지의 전체 문단
- 더하기+ 개별 아이템: 굵은 소제목부터 다음 소제목까지의 문단

### 링크 위치

- 외부 링크 없음 (link 필드는 빈 문자열)

## 추출 규칙

1. **상단 개요(❙ 기호 불릿 목록)는 본문과 중복이므로 추출하지 말 것.** 반드시 하단 본문 섹션에서 추출한다.
2. **심층 기사 섹션**(`깊게 보는 ...` 또는 `정치·사회 | ...`, `경제 | ...` 헤더):
   - 한 섹션당 독립 아이템 1개로 추출
   - 요약은 400~800자 분량으로 구체 수치·인물·배경을 충실히 담을 것
   - 인용문("...")과 고유명사(인물명, 기관명, 국가명)는 반드시 보존
3. **더하기+ 섹션**(`더하기+ 정치·사회 | ...`, `더하기+ 경제 | ...` 등):
   - 섹션 안의 굵은 소제목 각각을 독립 아이템으로 추출
   - 소제목 + 본문 문단을 합쳐 150~400자 요약으로 압축
   - 본문에 `...`, `(생략)`, `.`만 있는 줄, "생략" 단어만 있는 줄이 나오면 그 이후 잘린 아이템은 **버림** (VIP 전용 미리보기 컷오프)
4. 제외 영역 (아래 "제외 영역" 참조)

## 제외 영역

- 상단 개요 `❙` 불릿 목록 (본문과 중복)
- "VIP 독자는", "VIP 구독자 최소 ... 발송", VIP 구독 홍보 박스 전체
- "뉴스레터 수신", "수신거부", 미스터동 방향, 법적 고지, 광고/제휴/강연 섹션
- 푸터 주소·이메일·연락처
- 본문 중 `...생략...` 이후 잘린 콘텐츠

## 특이사항

- 메일 1통당 예상 아이템 수: 심층 기사 2개(정치·사회 + 경제) + 더하기+ 간추린 아이템 4~8개 ≈ **총 6~10개**
- 심층 기사 요약은 반드시 400자 이상. 1~2줄 요약은 금지 (상단 개요의 1줄 메타를 잘못 긁어온 것이 원인)
- 각 심층 기사에는 여러 개의 소제목(예: "버틸 수 없는 청구서", "파키스탄과 중국 그리고 한 장의 쪽지")이 있는데, 이는 한 기사 내 목차일 뿐이므로 **별도 아이템으로 쪼개지 말 것**
- 더하기+ 섹션은 VIP 미리보기이므로 첫 1~2개 아이템만 온전히 나오고 나머지는 "생략"으로 잘림 → 온전한 것만 사용
