# SKILL: MIT 테크놀로지 리뷰

MIT Technology Review 한국판 뉴스레터. 메인 기사 + 추천 기사.

---

## 메타 정보

```yaml
from_patterns:
  - "customer@technologyreview.kr"
  - "technologyreview.kr"
  - "MIT 테크놀로지 리뷰"
type: main_plus_recommendations
language: ko
frequency: weekly
```

## 추출 규칙

### 포함
| 유형 | 처리 |
|------|------|
| 메인 기사 | 반드시 추출 (핵심 콘텐츠) |
| 이 주의 추천 기사 | 소개 내용 있으면 추출 |

### 제외
| 유형 | 이유 |
|------|------|
| 지난주 인기 뉴스 | 페이월 + 소개 없음 |
| 인사말 | 뉴스 가치 없음 |
| 푸터 | 뉴스 가치 없음 |

---

## 메인 기사 추출

### 특징
- 인터뷰, 피처 기사가 많음
- 상세한 소개가 있음
- 종종 장문 (1,500자+)

### 추출 범위
```
시작: 인사말 직후 또는 기사 제목
종료: "▶ 자세히 읽기" 또는 "이 주의 추천 기사" 직전
```

### 장문 판단
```
메인 기사 소개가 1,500자 이상 → is_longform: true
(대부분의 메인 기사는 장문)
```

## 추천 기사 추출

### 조건
소개 내용이 있어야 추출:

```
[O] 추출:
[OPINION] 모두가 원하지만 아무도 가질 수 없는 AI 주권
한국 등 여러 국가가 'AI 주권'을 내세워 인프라 투자를 늘리고 있지만...
-> 소개 내용 있음 -> 추출

[X] 제외:
1. AI 에이전트의 미래
-> 제목만 있고 소개 없음 -> 제외
```

### 각각 별도 아이템
추천 기사는 각각 독립적인 주제이므로 **별도 아이템**으로 추출

## 지난주 인기 뉴스 - 제외

### 이유
1. 제목 + 링크만 있음 (소개 없음)
2. 페이월 콘텐츠
3. 클릭해도 구독자만 접근 가능

### 제외 신호
```
"지난주 인기 뉴스"
"지난 주 인기 기사"
"Most Popular"
"Top Stories"
숫자로 시작하는 제목 목록 (1. 2. 3.)
```

## 링크 처리

### 링크 텍스트
```
"▶ 자세히 읽기" → "MIT Technology Review 기사"
"▶ 읽어 보기" → 기사 제목 사용
```

### URL 정제
```
제거: utm_source, utm_medium, utm_campaign
유지: 기사 ID
```
