## 구조 분석

### 뉴스레터 특징
- 에세이/칼럼 형식의 단일 주제 뉴스레터
- 개인적인 경험담과 유머를 섞어 주제에 접근
- '출처 : MBC <무한도전>' 같은 이미지 캡션/출처 표기 포함
- '6월 농담 전문 보러가기 Click' 링크로 전체 기사 유도

### 아이템 경계
- 단일 주제이므로 별도 아이템 경계 없음
- 전체 본문이 하나의 아이템을 구성

### 제목 위치
- 이메일 제목(subject)이 곧 기사 제목: '🍔 시골도 문 앞에 쉑쉑버거가 올까? (스포: 안 옴)'
- clean_text 첫 줄에 동일 제목이 텍스트로 반복됨

### 본문 위치
- 제목 다음 줄부터 '6월 농담 전문 보러가기 Click' 링크 직전까지의 전체 텍스트
- 개인적 에세이, 이미지 설명, 질문 형식의 문장들로 구성

### 링크 위치
- 본문 말미의 '[6월 농담 전문 보러가기 Click](URL)' 패턴
- URL은 stibee 트래킹 링크 또는 nongdam.kr 직접 링크

## 추출 규칙

1. subject에서 이모지를 제거하여 제목 추출
2. 제목 다음 줄부터 '전문 보러가기' 링크 이전까지의 모든 텍스트를 본문으로 취합
3. '출처 :', '©', '수신거부', '매거진 농담' 등 꼬리말 이전까지를 본문 범위로 설정
4. '전문 보러가기' 링크를 item.link로 사용
