# SKILL: 범용 뉴스레터

새로운 뉴스레터 또는 매칭되는 SKILL이 없는 뉴스레터에 적용하는 범용 처리 규칙.

---

## 메타 정보

```yaml
from_patterns: []  # 범용이므로 패턴 없음
type: generic
priority: lowest   # 다른 SKILL에 매칭되지 않을 때만 사용
```

---

## 적용 조건

다음 조건에서 이 SKILL 적용:
1. `from` 필드가 어떤 뉴스레터 SKILL의 `from_patterns`에도 매칭되지 않음
2. 새로 추가된 뉴스레터

---

## 구조 분석 전략

### 1단계: 헤더/푸터 식별
```
- 첫 5줄: 대부분 헤더 (날짜, 호수, 구독 안내)
- 마지막 10줄: 대부분 푸터 (수신거부, 저작권, 공유)
```

### 2단계: 본문 섹션 식별
```
신호들:
- 이모지로 시작하는 줄 (🎯 🦾 🤖 👉 📌)
- 숫자로 시작하는 줄 (1. 2. 3.)
- 굵은 표시 (**텍스트**)
- 빈 줄로 구분된 문단
```

### 3단계: 아이템 경계 판단
```
- 새로운 주제가 시작되면 새 아이템
- 같은 주제의 연속이면 같은 아이템
- 불확실하면 하나의 아이템으로 통합
```

---

## 제외 패턴 (공통)

다음 내용은 아이템으로 추출하지 않음:

### 뉴스레터 구조
- "웹에서 보기"
- "구독하기", "구독 취소"
- "지난호 보기"
- "뉴스레터 공유"
- "Copyright ©"
- "All rights reserved"

### 소셜 버튼
- "[이미지: 트윗하기]"
- "[이미지: 공유하기]"
- "[이미지: 페이스북]"

### 마케팅
- "지금 구독하세요"
- "무료 체험"
- "할인 혜택"

### 기타
- 이메일 수신 관련 안내
- 개인정보 처리방침
- 피드백 요청 ("의견을 남겨주세요")

---

## 추출 기준

### 포함
- 뉴스/정보 콘텐츠
- 분석/인사이트
- 인터뷰/칼럼
- 트렌드 리포트
- 이벤트 (정보 가치 있는 경우)

### 제외
- 순수 광고 (제품 판매만)
- 구독 유도
- 커뮤니티 안내

## 아이템 구조화

```json
{
  "items": [
    {
      "title": "본문에서 추출한 핵심 제목",
      "summary": "300~500자 요약",
      "keywords": ["키워드1", "키워드2", "키워드3"],
      "link": "원문 URL (없으면 빈 문자열)",
      "source": "뉴스레터 이름"
    }
  ]
}
```

---

## 개선 요청

새 뉴스레터가 자주 등장하면:
1. 해당 뉴스레터의 구조 분석
2. 전용 SKILL 파일 자동 생성 (adaptive_learning.js)
3. 카탈로그에 등록 (newsletters.json)
