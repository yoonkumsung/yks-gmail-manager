## 구조 분석

### 뉴스레터 특징
- 첫 번째 아이템은 부국장의 긴 에디토리얼 (메인 기사)
- 이후 짧은 칼럼/뉴스 아이템이 이어짐
- 모든 아이템은 "전문보기" 링크로 종료

### 아이템 경계
- 아이템 종료: "전문보기" 텍스트
- 첫 번째 아이템: "안녕하세요" 인사말부터 첫 번째 "전문보기"까지
- 이후 아이템: 대괄호 섹션명(예: [횡설수설/김창덕]) 또는 볼드 제목으로 시작, "전문보기"로 종료

### 제목 위치
- 첫 번째 아이템: 메일 제목 또는 본문 첫 문장에서 추출
- 이후 아이템: 대괄호 섹션명(예: [횡설수설/김창덕]'학점 인플레' 제동 건 하버드) 또는 볼드 텍스트

### 본문 위치
- 제목/인사말 이후부터 "전문보기" 직전까지의 텍스트

### 링크 위치
- "전문보기" 텍스트에 포함된 URL

## 추출 규칙

**핵심: "전문보기"가 나올 때마다 하나의 아이템이 끝난다. 텍스트에서 "전문보기"를 세면 아이템 수를 알 수 있다.**

1. 텍스트를 "전문보기" 기준으로 분할
2. 첫 번째 블록 = 메인 에디토리얼 ("안녕하세요"부터 첫 "전문보기"까지)
3. 이후 각 블록 = 독립 아이템 (제목 행 → 본문 → "전문보기")
4. 각 아이템 제목: 볼드(**텍스트**) 또는 대괄호([섹션명/필자]텍스트) 형태
5. "동아일보 칼럼을 통해 본 오늘, 세상" 같은 섹션 구분 텍스트는 제목이 아님, 건너뛰기

### 추출 예시
입력에 "전문보기"가 7번 나오면 → 7개 아이템 추출
```
아이템1: 안녕하세요~삼성전자 노조... → 전문보기
아이템2: 정원오, 구의역 참사... → 전문보기
아이템3: 국힘, '김태흠 모두발언 삭제'... → 전문보기
...
```
