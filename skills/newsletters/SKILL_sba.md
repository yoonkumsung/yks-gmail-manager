## 구조 분석

### 뉴스레터 특징
- 이모지와 카테고리명으로 구분된 다중 섹션 구조
- 각 섹션 내에서 날짜(~MM/DD)가 표시된 여러 지원 사업/공고 항목을 나열
- 각 항목은 제목과 신청 마감일만 포함하며 상세 설명이 없음

### 아이템 경계
- 각 아이템은 줄바꿈으로 구분된 단일 라인
- 아이템 라인은 이모지 섹션 헤더('📗 지식재산권', '💡오픈이노베이션' 등) 아래에 위치
- 아이템 라인은 사업명과 마감일(~MM/DD) 형식

### 제목 위치
- 각 아이템의 전체 라인이 제목 역할을 함 (예: 'K-브랜드 해외 권리화 지원(~4/24)')

### 본문 위치
- 본문 텍스트가 없음. 각 아이템은 제목/마감일 정보만으로 구성됨

### 링크 위치
- 개별 아이템에 대한 직접적인 링크가 제공되지 않음. 'SBA 사업 더보기 +'라는 일반적인 링크만 존재

## 추출 규칙

1. 이모지로 시작하는 섹션 헤더('📗', '💡' 등)를 식별
2. 각 섹션 헤더 아래, 다음 섹션 헤더나 'SBA 사업 더보기 +' 라인 이전까지의 모든 줄을 후보 아이템으로 간주
3. 후보 아이템 라인에서 날짜 형식(~MM/DD)을 포함하는 라인만 최종 아이템으로 추출
4. 날짜 형식이 없는 라인(예: '지식재산권 보호 심판·소송 지원')은 제외
