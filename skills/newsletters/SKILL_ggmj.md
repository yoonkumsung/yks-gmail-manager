## 구조 분석

### 뉴스레터 특징
- 각 아이템은 대괄호([])로 시작하는 제목 라인으로 구분됨
- 제목 뒤에 본문이 이어지고, 마지막에 '신청하기' 또는 '문의하기' CTA 링크가 위치
- 아이템 사이에는 구분선(테이블 형태의 | | | |)이 있음
- 하단에 프로모션, 콘텐츠, 채용공고 등 추가 섹션이 있으나 아이템 추출 대상 아님

### 아이템 경계
- 대괄호([])로 시작하는 제목 라인 (예: '[기고만장] HRer! 2026 우리 지금 만나! (지역별 담당자 모임)')
- 각 아이템 종료 후 구분선(테이블 형태의 | | | |)이 나타남

### 제목 위치
- 대괄호([]) 안의 텍스트 전체 (예: '[기고만장] HRer! 2026 우리 지금 만나! (지역별 담당자 모임)')

### 본문 위치
- 제목 라인 다음 줄부터 '신청하기' 또는 '문의하기' 텍스트 직전까지의 모든 텍스트

### 링크 위치
- '신청하기' 또는 '문의하기' 텍스트에 연결된 URL (clean_text에는 URL이 명시되지 않아 추출 불가)

---

## 추출 규칙

1. 대괄호로 시작하는 라인을 찾아 제목으로 추출 (대괄호 포함 전체)
2. 제목 다음 줄부터 '신청하기' 또는 '문의하기'가 나오기 전까지를 본문으로 추출
3. '신청하기' 또는 '문의하기' 텍스트를 링크로 간주하나, URL이 없으면 빈 문자열 처리
4. 하단의 프로모션, HR 콘텐츠, 채용공고, 뉴스레터 다시보기 섹션은 제외
