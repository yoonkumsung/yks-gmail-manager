## 근본 원인 (우선순위 · 계층)

### P0 — 핵심원칙 위반

**RC-O★ 할루시네이션(날조) — 총 ~80+건 확정**
- 패턴: **빈약 입력**(SeekingAlpha 헤드라인-only, 크롤 실패로 빈 페이지, 본문 0 메일, 청크 빈 조각) → LLM이 자체지식으로 가짜 단신/수치/인물/판결 생성 → **병합 통과해 최종 리포트 생존**.
- 집중처: 파손 라벨(투자·경제·시사) + 고볼륨 꼬리(NYT_시사). 정상 라벨+충실 본문은 **0건**.
- 대책(LLM+RULE): ① 프롬프트 "본문/원문 없으면 해당 아이템 요약 생성 금지, 빈 슬롯 채우지 말 것" ② **추출 후 검증 룰**: 요약의 핵심 고유명사/수치가 원문 clean_text에 없으면 드롭/플래그 ③ 크롤 실패 시 teaser만 보존(창작 금지).

**RC-M 누락**
- NYT "헤드라인+By저자/Read more" 연결포맷 분리 실패 → 메일 통째 0추출(17뉴스). 투자 날조가 실제 메인기사 슬롯 차지(8건). careet "1분 요약" 블록 미추출(4건).
- 대책: RULE(파싱 분리 보강) + SKILL(단신 블록 명시) + RC-O 가드(날조로 인한 누락 동시 해소).

**RC-B/R 병합 dedup 저재현율 (전 라벨 공통, 최다 빈도)**
- teaser+크롤전문 2~3중 추출 + 청크 중복 + **link 무결성 붕괴**로 같은 사건이 대거 잔존(라이프 한강 12, 인문학 8, NYT_라이프 프라임데이 5, 미국_시사·중국_시사 90 등). 오병합(다른 사건 합침)은 거의 0 — **방향은 맞으나 재현율이 낮음**.
- 대책(RULE): ① **동일 link 정규화 사전병합**(P1-3) ② article-id/message_id+저자 시그니처 dedup ③ 제목 유사도 보강. 이걸로 LLM 병합 호출도 대폭↓(비용·시간).

**RC-D 정크**: 광고 랜딩·티저·영상 발화자별 분해·시세위젯·유료벽 과거글·게임링크·푸터 바이라인 오추출. **단 <50자 대부분은 정상 단신** — 길이 기준 오탐 주의(콘텐츠 판단 필요). → RULE 선차단(스폰서/푸터/소셜/게임 블록) + 추출단위 가드.

## LLM vs RULE 분담 (수정 시)
- **RULE/CODE**: link 정규화·dedup, enrich/메타, 정크 블록 차단, charset, 파싱 분리, 스키마 복구, 라벨 재분류, 추출후 앵커 검증.
- **LLM**: 빈약입력 날조 억제(프롬프트), 의미 병합 잔여, 번역/음역, 시제·사건분리.
