### 근본원인(RC) 분류
**룰베이스로 풀 문제 (코드·JSON·필터):**
1. **중복 병합 강화** — URL 정규화 + 동일출처(같은 LINK/도메인) 1차 dedup 후 LLM 의미병합. MIT·더밀크에서 병합이 0건인 것은 명백한 버그 신호.
2. **정크 필터** — selectstar.ai 제품 랜딩페이지, stock.mk 시세위젯/인기리스트, Ghost "Keep reading"/유료벽 블록, 웨비나 배너를 발신자·URL 패턴으로 추출 전 컷.
3. **출처 LINK 바인딩** — 청크-URL 결정적 매핑(MIT 7건·TechCrunch 다수 오매핑 교정).
4. **"유료 전용이라 확인 불가" / "~를 다룬 기사" 메타서술** — 금지표현 정규식 필터로 자동 폐기.
5. **본문 미확보 링크(Recent Articles 등)** — 본문 크롤 실패 시 드롭(메타서술·할루 부연의 원천).
6. **종료 행사 시점** — 날짜 파싱 + "모집종료/Application Closed" 시그널 룰.

**LLM으로 풀 문제 (프롬프트·SKILL):**
1. **사건 분리 시 환각** — 한 기사를 조각낼 때 빈 조각을 자체지식으로 메우는 패턴(19ef5007 #13·#14). 프롬프트에 "분리된 조각에 정보 없으면 생성 금지" 강화.
2. 시제 정확성(추가 예정 vs 완료), "출시" 과장 프레이밍 완화.
3. 번역 표기 일관성(길모/기예모, Klue 업종 오역).

### 발신자별 baseline 메모 (파손 라벨 비교용 기준)
- **모범:** 요즘IT(전건 무결), 더밀크(구조 깨끗, 누락0·할루0·정크0, 문제는 중복뿐), TechCrunch(누락0·번역 양호, 광고 배제 우수).
- **구조적 취약:** AI코리아(Ghost 사이드바·유료벽 → 실뉴스 1건이 30여건으로 팽창), 셀렉트스타(홍보메일 → 9건 중 8건 정크), MIT(병합 미작동 → 11기사 중복), 미라클(stock.mk 위젯 오추출), 바이라인(원문 섹션 과분할 + 본문없는 메타서술).

> **결론:** IT는 LLM 추출·번역·요약 품질 측면에서 신뢰할 만한 baseline이다. 파손 라벨에서 **누락/번역/할루가 IT보다 나쁘다면 그것은 라벨 에이전트 파손 고유 문제**로 귀속할 수 있다. 반대로 **중복·정크는 IT에서도 이미 취약**하므로 파손 라벨에서 동일 증상이 보여도 라벨 파손 탓으로 단정해선 안 되며 공통 룰베이스 결함으로 봐야 한다.
